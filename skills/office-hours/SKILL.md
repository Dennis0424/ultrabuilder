# Office Hours — Product Interrogation

## When to Invoke

Use BEFORE any planning begins. This challenges the fundamental framing of what you're building. Different from CEO review — this happens before a plan exists and questions whether you're even asking the right question.

## Two Modes

Ask the user which mode they want:

- **Startup Mode** — uncomfortable questions, challenges premises hard, pushes toward what users actually need vs. what they say they want
- **Builder Mode** — enthusiastic collaborator, helps refine and sharpen, assumes the direction is roughly right

## The 6 Forcing Questions

Ask one at a time via AskUserQuestion. Wait for each answer before proceeding.

### 1. Who exactly feels this pain?

Not "users" — a specific person with a name and a day. What are they doing when they hit the wall? What do they do instead right now?

### 2. What happens if we never build this?

Force the "do nothing" option into the open. If the answer is "nothing much changes," that's a signal.

### 3. What would make this a 10x solution (not 10% better)?

Push past incremental. What would make someone text their friend about this? What would make them angry if you took it away?

### 4. What's the fastest way to test if this matters?

Not build. TEST. What's the cheapest signal that real humans want this? Can we test it in a day? An hour?

### 5. What are we afraid to admit about this?

The uncomfortable truth. The thing that makes this harder than it looks. The reason the last attempt failed. The political reason this is being proposed.

### 6. If we could only keep one thing, what is it?

Strip everything else away. What's the irreducible core? If you only shipped this one thing, would it still matter?

## After the Questions

Produce a brief (under 200 words) that captures:

- **The real problem** (reframed from the forcing questions)
- **The core bet** (the one assumption that must be true for this to work)
- **Suggested next step** (often: test the bet cheaply before building)

## Principles

- This is NOT a planning tool — it's a thinking tool
- Uncomfortable questions are the point
- "I don't know" is a valid answer and often the most honest one
- If all 6 answers come back confident and clear, the idea is ready for `/build-direction`
- If answers are uncertain, suggest testing before building
